Documentation page

This is a server-side component of a “movies” web application which is similar to Netflix. This web application will provide users with access to information about different movies as requested.

HTTP Method Request URL Request body data format Response body data format
GET Get all movie lists '/movies' None Get data array of all movie lists in a JSON format
GET Get data about a single movie title '/movies/:title' None Get an array of a single movie data in JSON format, based on title entered in the URL parameter. Example: { title: 'Inglorious Basterds', director: ['Quentin Tarantino'], genre: ['Action'], yearOfRelease: 2009 }
GET Get movie by genre '/genres/:genre' None Get an array of movie by genres in a JSON format
GET Get data about a movie director 'directors/:director' None Get an array of movie director/directors data in a JSON format, based on the director entered in the URL parameter
POST Allow clients to create account '/users/:newUser' user information is added to the server in a form style A text message of the client's added information, included with a unique id created by uuid v4 and an extra success message
PUT Allow clients to update their client info '/users/:userName' None A text is returned co confirm the updated information in this case the username
POST Allow clients to add their favourite movies to their library '/movies/:addFavourite' An array of JSON data containing the added movie. Example: { title: 'Jungle Book', director: ['Jon Favreau'], genre: ['Family'], releaseYear: 2016 } An array of the added movie data in JSON format, with a text confirming added data and an added ID. Example: { id: 79445, title: 'Jungle Book', director: ['Jon Favreau'], genre: ['Family'], releaseYear: 2016 }
DELETE Allow clients to remove a movie from their list of favorites '/movies/:movieTitle' None Returns a text confirming which movie that was actually deleted
DELETE Allow clients to terminate account '/users/:username' None Returns a text confirming user deregistration
